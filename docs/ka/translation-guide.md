# თარგმანში მონაწილეობა

გთხოვთ თავიდან გაეცნოთ [საერთო წესებს](https://developer.mozilla.org/en-US/docs/MDN/Guidelines), კერძოდ, გვერდიდან ["Writing style guide"](https://developer.mozilla.org/en-US/docs/MDN/Guidelines/Writing_style_guide).

ძირითადი თემა — [MDN-ის თარგმნა ქართულ ენაზე](https://github.com/mdn/translated-content/issues/276)

## თარგმანის რეკომენდაციები

- გამოიყენეთ ტირე დეფისის ნაცვლად (ის უფრო გრძელი უნდა იყოს ვიდრე დეფისი). ტირე: «`—`», დეფისი: «`-`».
- გამოიყენეთ ბრჭყალები («»), გარდა კოდების მაგალითებისა.
- თარგმნეთ კოდების მაგალითები, შეცდომების გარდა.
- გამოიყენეთ ტერმინოლოგია [ლექსიკონი «ვებ სტანდარტების»](https://github.com/web-standards-ka/dictionary), ასევე გადახედეთ ქვემოთ მოცემულ ლექსიკონს.
- ნუ თარგმნით კომპანიების სახელებს, ბიბლიოთეკებს და ა.შ.
- ნუ თარგმნით აბრევიატურებს (API, DOM, UI).
- აპოსტროფით უცხო სიტყვებს ნუ დაამახინჯებთ. თუ ეს საჭიროა, სცადეთ სიტყვის გაქართულება (URL`ები -> URL-მისამართები).

## ტერმინების ლექსიკონი

**გთხოვთ, გლოსარიუმი შეინახოთ ანბანური თანმიმდევრობით.**

| ორიგინალური ტერმინი | თარგმანი |
| ------------------ | ---------- |
| cache | ქეშ |
| feature | შესაძლებლობა (_ნაკლებად სასურველი სინონიმი: ფუნქციონირება_) |
| functionality | ფუნქციონირება (ან კონტექსტიდან გამომდინარე — _შესაძლებლობა_) __*__ |
| general knowledge | ზოგადი ინფორმაცია (როდესაც გამოიყენება როგორც სათაური) |
| learn more | დამატებითი რესურსები (როდესაც გამოიყენება როგორც სათაური) |
| listener | დამმუშავებელი (მოვლენები) |

| production | წარმოება |

| technical reference | ტექნიკური მასალები (როდესაც გამოიყენება როგორც სათაური) |

__*__ — უმჯობესია თავიდან აიცილოთ პირდაპირი თარგმანი, როდესაც ეს შესაძლებელია, სინონიმების გამოყენებით ან მნიშვნელობის უკეთ ახსნით.
